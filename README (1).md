* **⚠️ 에러 노트:** 파이썬은 대소문자를 엄격하게 구분하므로 `print()`와 `Print()`는 완전히 다르게 인식됩니다. 확장자는 언제나 `.py`인 점을 명심합니다.

### 📦 02. 변수 맵핑과 산술 수식 (Variables & Operators)
* **이름표로서의 변수:** 파이썬의 변수는 값을 직접 담는 상자가 아니라, 메모리에 생성된 객체의 주소를 가리키는 '참조(Reference)' 구조임을 깨달았습니다.
* **자동 자료형 생성:** 정수, 실수, 문자열 등 자료형을 미리 선언하지 않아도 대입문(`=`)을 통해 변수가 자동으로 생성됩니다.
* **데이터 캐스팅:** `input()` 함수로 읽어오는 유저 입력값은 항상 문자열이므로, 산술 연산을 하려면 `int()`나 `float()`로 명시적 형 변환을 거쳐야 합니다.

```python
x, y = 100, 200
result = x + y
print(result) # 300

# 연산자 툴킷: +, -, *, /, //(몫), %(나머지), **(거듭제곱)
num = int(input("숫자를 입력하세요: "))
```

### 🛣️ 03. 프로그램의 분기, 조건문 (Conditional Statements)
* **실행 흐름 제어:** 무조건 일직선으로만 가던 프로그램에 `if`, `elif`, `else` 구문으로 상황에 따른 아름다운 분기점을 만들어 주었습니다.
* **논리적 조합:** 비교 연산자(`>`, `<`, `==`, `!=` 등)와 두 개 이상의 조건을 엮어주는 논리 연산자(`and`, `or`, `not`)의 우선순위를 실습했습니다.
* **📏 인덴트 규칙:** 파이썬은 중괄호(`{}`) 대신 **들여쓰기(Indentation)**로 코드의 영역을 구분하므로 정렬 스페이스바 규칙을 철저히 지켜야 합니다.

```python
score = 80
if score >= 60:
    print("합격")
else:
    print("불합격")
```

<br>

### 🔄 04. 무한한 연산, 반복문 (Loop Statements)
* **자동화의 핵심:** 인간이 기계보다 뒤처지는 영역인 '단순 반복 작업'을 `for`문과 `while`문으로 제어하는 방법을 익혔습니다.
* **수열 제어판:** `range()` 함수를 활용하여 루프의 인덱스 시퀀스를 유연하게 생성하는 기술을 배웠습니다.
  * `range(5)` $ightarrow$ 0부터 4까지 총 5회 순회
  * `range(1, 6)` $ightarrow$ 1부터 5까지 순회
  * `range(1, 10, 2)` $ightarrow$ 1부터 9까지 2칸씩 점프하며 순회 (1, 3, 5, 7, 9)
* **루프 응용:** 중첩 반복문을 통해 다차원 데이터를 처리하고, 누적 합산 로직을 구현했습니다.

```python
# while 루프 제어 (무한 루프 방지를 위한 탈출 플래그 관리)
count = 0
while count < 5:
    print(count)
    count += 1
```

<br>

### 🛠️ 05. 기능의 묶음, 함수 (Functions)
* **모듈화의 첫걸음:** 반복해서 작성하는 코드 조각을 하나의 가방(`def`) 안에 포장하여 프로그램의 재사용성을 극대화했습니다.
* **데이터 파이프라인:** 값을 던져주는 **매개변수(Parameter)**와 연산 처리가 완료된 결과물을 바깥으로 가로채 반환하는 **`return`**의 메커니즘을 마스터했습니다.

```python
def add(x, y):
    return x + y # 함수 밖으로 결과 전달
```

<br>

### 📊 06. 유연한 선형 구조, 리스트 (List)
* **시퀀스 컬렉션:** 순서가 있고 언제든 데이터를 넣고 뺄 수 있는(Mutable) 가장 대표적인 자료구조입니다.
* **인덱스 트릭:** 0부터 시작하는 인덱싱은 물론, 뒤에서부터 거꾸로 접근하는 음수 인덱스(`-1`) 시스템을 배웠습니다.
* **내장 인터페이스 유틸리티:**

| 데이터 조작 (`append`, `insert`, `remove`, `pop`, `sort`) | 데이터 통계 (`len`, `max`, `min`, `sum`) |
| :--- | :--- |
| 리스트 뒤나 원하는 위치에 원소를 추가/수정하고 특정 값이나 위치의 요소를 삭제 및 오름차순 정렬 수행 | 시퀀스 데이터의 총 원소 개수, 최댓값, 최솟값 및 전체 합계를 구하는 빌트인 함수 활용 |

```python
# 리스트 슬라이싱 기법 [시작:끝+1]
a = [0, 1, 2, 3, 4]
print(a[1:3]) # [1, 2] 추출
print(a[2:])  # [2, 3, 4] 끝까지 추출
```

<br>

### 🗂️ 07. 맞춤형 데이터 주머니, 컬렉션 (Collections)
각 데이터의 특성과 목적에 따라 선택해야 하는 다양한 컨테이너들의 차이점을 확실히 비교 정리했습니다.

* **튜플 (`Tuple`):** 한 번 생성하면 값을 바꿀 수 없는 불변(Immutable)의 리스트. (요소가 단 1개일 땐 `(item,)` 처럼 쉼표를 꼭 찍어야 하는 예외 규칙 숙지)
* **세트 (`Set`):** 순서가 중요하지 않고, 오직 '중복 없는 고유값'만 남기고 싶을 때 사용하는 집합형 구조.
* **딕셔너리 (`Dictionary`):** 단어를 찾듯 **Key-Value** 쌍으로 데이터를 매핑하여 원하는 값을 초고속으로 탐색하는 해시 기반 구조.
* **문자열:** 문자열 역시 글자들의 시퀀스형 배열이기 때문에 리스트처럼 인덱싱과 슬라이싱이 완벽하게 호환됨을 이해했습니다.

```python
# 딕셔너리로 구현한 연락처 매핑
phone = {"Kim": "010-1234-5678", "Lee": "010-1111-2222"}
print(phone["Kim"]) # Key를 통해 데이터 Direct 조회
```

### 🏗️ 08. 클래스와 도면 설계 (Object-Oriented Programming)
* **OOP 패러다임:** 데이터(속성)와 행동(메서드)을 하나로 묶어 현실 세계의 사물을 코드로 추상화하는 객체지향 프로그래밍에 입문했습니다.
* **설계도와 인스턴스:** 객체가 메모리에 찍혀 나올 때 최초로 자동 트리거되는 생성자 `__init__()` 메서드와 인스턴스 나 자신을 명시하는 마법의 키워드 `self`를 배웠습니다.

```python
class Car:
    def __init__(self, model, color):
        self.model = model   # 객체의 필드 초기화
        self.color = color
        self.speed = 0

    def speed_up(self):      # 객체의 행동 정의
        self.speed += 10

# 클래스를 설계도로 삼아 자동차, TV, 사람 등 다양한 객체를 직접 빌드해 보았습니다.
```
